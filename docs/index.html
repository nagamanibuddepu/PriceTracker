What it does

Core Features

🔍

Real-Time Price Comparison

Search any product and see results from multiple e-commerce platforms side by side. No tab-switching. No manual searching.

⚙️

Distributed Scraping Pipeline

Jobs are offloaded to BullMQ background workers via Redis queues — preventing API timeouts and keeping the UI fully responsive.

🤖

AI Shopping Assistant

Powered by Google Gemini. Answers product questions, compares specs, and suggests alternatives with contextual shopping awareness.

🔗

Product Title Normalization

Product titles vary wildly across platforms. The system normalizes them to match identical items across different marketplaces.

📈

Price History Tracking

Price data is stored over time in MongoDB. Trends are visualized so users can spot the best moment to buy.

🛡️

IP-Based Rate Limiting

Public demo endpoints are rate-limited per IP to protect external API quotas and prevent abuse in a scraping-heavy public deployment.

System design

Architecture

request lifecycle

User Search
│
Next.js Frontend — polls job status
│
Next.js API Routes — returns job ID immediately
│
BullMQ Queue — Redis-backed async queue
│
Background Worker — separate long-running process
├─ Puppeteer  ·  Cheerio  ·  SerpAPI  ·  ZenRows  ·  ScrapingBee
│
MongoDB Atlas — cache + price history
│
Results → UI

Scraping is slow and unpredictable. Putting scraping work directly in an API route would time out within seconds. By offloading to a BullMQ queue processed by a dedicated worker, the API returns immediately with a job ID, and the frontend polls for completion — the same pattern used by production ETL and scraping systems.

Technology choices

Tech Stack & Rationale

Layer Technology Why this choice
Frontend Next.js (App Router) SSR + client routing in one framework. App Router enables layout-level data fetching and co-located API routes without a separate Express server.
UI Shadcn UI + Radix UI Accessible, unstyled primitives with full design control. Radix handles the hard parts (focus traps, keyboard nav) without forcing visual opinions.
Styling Tailwind CSS Utility-first approach eliminates CSS file sprawl on a fast-iteration side project. No build-time surprises.
Animation Framer Motion Declarative animation API. Used for job-status transitions and result reveal animations where CSS alone gets unwieldy.
Job Queue BullMQ Redis-backed job queue purpose-built for Node.js. Handles retries, delays, and concurrency control out of the box — replacing complex manual polling logic.
Cache + Broker Redis (Upstash) Fast ephemeral store for BullMQ jobs. Upstash is serverless-compatible Redis — works on Vercel's edge environment for rate limiting without a dedicated Redis server.
Database MongoDB Atlas Product data schemas differ per e-commerce source. A document store avoids painful schema migrations as scrapers evolve and new fields appear.
Scraping Puppeteer, Cheerio, SerpAPI, ZenRows, ScrapingBee, RapidAPI Multi-source redundancy. Puppeteer for JS-rendered pages; Cheerio for fast static HTML parsing; paid scraping APIs as fallback and for anti-bot protection.
AI Google Gemini API Generous free tier. Strong instruction-following for the shopping assistant use case. Contextually aware responses for product comparison queries.
Deployment Vercel + Render/Railway + Upstash + Atlas Long-running worker processes can't run on Vercel's serverless functions (60s max). Worker is deployed separately on Render/Railway as a persistent service.
Key decisions

Engineering Highlights

Distributed scraping with BullMQ + Redis — gives retry logic, job priorities, and concurrency control out of the box. Prevents API routes from blocking on slow scrapers.
Multi-source scraping strategy — using several APIs (SerpAPI, ZenRows, ScrapingBee) in parallel increases coverage and reduces single-point-of-failure risk from any one source getting blocked.
MongoDB for heterogeneous product data — scraped product schemas differ per source. A document store avoids painful schema migrations as scrapers are added or modified.
Worker deployed separately from frontend — Next.js on Vercel runs as serverless functions with a 60s timeout ceiling. Background scraping workers need persistent processes; Render/Railway provides that separation cleanly.
Upstash Redis for serverless rate limiting — standard Redis can't run in Vercel's edge environment. Upstash is a REST-compatible Redis that works in both edge and Node.js contexts.

Project Description

PriceTracker — AI-Powered Price Comparison Platform
Built a full-stack price aggregation platform using Next.js, BullMQ, Redis, and MongoDB. Designed a distributed background scraping pipeline that decouples long-running jobs from API routes using a Redis-backed job queue, preventing timeouts under real-world scraping latency. Integrated Google Gemini as a contextual AI shopping assistant. Implemented IP-based rate limiting for public demo safety. Deployed frontend on Vercel with a separate long-running worker on Render, following production separation-of-concerns architecture.
